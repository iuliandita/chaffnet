# Security policy

## Supported versions

chaffnet is early alpha. Security fixes are provided for the latest tagged release. The `main`
branch is development code and may change without notice.

## Reporting a vulnerability

Use **Report a vulnerability** in this repository's GitHub Security tab. Do not disclose
vulnerability details in a public issue, discussion, or pull request.

Include, when available:

- the affected version or commit;
- the affected component and deployment mode;
- reproduction steps or a minimal proof of concept;
- the expected impact and required attacker access;
- any suggested mitigation.

If GitHub private vulnerability reporting is unavailable, email
[iulian.dita@gmail.com](mailto:iulian.dita@gmail.com). Do not include vulnerability details in a
public issue.

The project will acknowledge reports as capacity allows, validate the issue, coordinate a fix and
release, and credit reporters who want attribution. No bug bounty is currently offered.

## Scope notes

- Reports about the self-hosted server, hosted API mode, official container, TypeScript SDK, and MCP
  server are in scope.
- Spam or slop classification errors without a security impact should use the regular issue tracker.
- Vulnerabilities in third-party services or dependencies should also be reported to their upstream
  maintainers when appropriate.
